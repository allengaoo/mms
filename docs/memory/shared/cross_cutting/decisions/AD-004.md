# AD-004 · Kafka Avro + Schema Registry

## 决策
Kafka 消息统一使用 Avro 格式 + Confluent Schema Registry，替代原 JSON 格式。

## 背景
JSON 无 schema 约束，字段变更无法检测，导致消费者静默处理错误格式数据。

## 替代方案对比
| 方案 | 优劣 |
|:---|:---|
| A. JSON | 无 schema 保证，字段变更静默失败 |
| B. Protobuf | 更紧凑，但工具链复杂 |
| **C（选择）** | Avro + Schema Registry（fastavro 成熟，Confluent 生态兼容）|

## 影响
- 新增 Avro schema 字段必须有 `"default": null`（BACKWARD 兼容，见 MEM-L-006）
- 消费者用 `fastavro.parse_schema` 解析 schema
