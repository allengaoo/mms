# AD-005 · SQLAlchemy 事务策略二选一

## 决策
后端写操作必须选且仅选 Strategy A（begin-first）或 Strategy B（autobegin + explicit commit），严禁混用。

## 背景
`AsyncSession` 默认 `autobegin=True`；在 `execute()` 后调 `session.begin()` 导致 `InvalidRequestError`，在生产中造成 500 错误（见 MEM-DB-002）。

## 策略定义
```python
# Strategy A：读写混合 + 需要原子性
async with session.begin():
    result = await session.execute(...)
    session.add(obj)

# Strategy B：简单写入（已有 autobegin）
obj = await session.get(Model, id)
obj.field = value
await session.commit()
```

## 选择指南
- Strategy A → 优先（自动 rollback，更安全）
- Strategy B → 简单单表写入时使用
- ❌ 禁止在同一函数中混用两种策略
