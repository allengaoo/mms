# AD-008 · 知识索引三层架构（EP-099）

## 决策
引入三层知识索引（global-constraints / Domain Manifests / Deep Knowledge），通过 `MASTER_INDEX.md` 路由。

## 背景
EP 数量超过 99 个后，每次新 EP 需手工 @mention 10+ 个文件；约束分散导致模型跟随性差。

## 替代方案对比
| 方案 | 优劣 |
|:---|:---|
| A. 继续手工 @mention | 维护成本高，效率低 |
| B. 合并为一个超大 alwaysApply 文件 | token 消耗过高，失焦 |
| **C（选择）** | 分层索引，按需加载，global-constraints 仅 18 条红线 |

## 影响
- 所有新 EP 使用起手式：`@MASTER_INDEX + @SESSION_HANDOFF + @{domain}.md`
- EP-107 进一步演进为 `MEMORY_INDEX.json` 推理式检索（无需手工 @mention）
