# AD-007 · NullSafeNormalizer 作为 Kafka 发送前强制门

## 决策
在 `infrastructure/connector/normalizer.py` 实现 `NullSafeNormalizer`，作为所有数据源到 Kafka 的唯一归一化层。

## 背景
PostgreSQL、MySQL、S3 等不同数据源的原生 Python 类型在 fastavro 序列化时静默失败（见 MEM-L-002）。

## 替代方案对比
| 方案 | 优劣 |
|:---|:---|
| A. 各 source adapter 内部各自转换 | 分散，不一致 |
| B. 在 Kafka Producer 内部转换 | 侵入 MQ 层，违反架构边界 |
| **C（选择）** | 独立 normalizer 层，Worker 层调用，注册器支持扩展 |

## 影响
- 所有 Ingestion Worker 发 Kafka 前必须调 `normalize_record()`
- 新增数据源只需在注册器中添加类型处理器，无需修改 Worker
