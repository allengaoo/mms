# ENV-001 · K8s 生产模拟环境部署快照（OrbStack）

## WHERE（适用场景）

- 编写涉及基础设施的 EP（后端启动、DB 迁移、数据管道配置）
- 调试数据库/消息总线/向量库连接问题
- 设计数据流向（Control Plane ↔ Data Plane）
- 评估新功能对哪些基础设施组件有依赖

## HOW（核心配置）

### 集群概况

| 属性 | 值 |
|:---|:---|
| 运行环境 | **OrbStack** 单节点 K8s（模拟生产集群） |
| 应用命名空间 | `mdp` |
| 控制平面地址 | `https://127.0.0.1:26443` |
| 后端 Service | `mdp-backend.mdp.svc.cluster.local:8000` |
| 前端 Service | `mdp-frontend.mdp.svc.cluster.local:80` |
| 当前后端镜像 | `mdp-backend:ep113`（2026-04-14 更新） |
| 当前前端镜像 | `mdp-frontend:ep104` |

### 元数据库 — MySQL（系统核心）

```
用途:  MDP 系统元数据库（meta_* / sys_* 表）
Pod:   mysql-0（StatefulSet，mysql:8.0-debian）
集群内: mysql.mdp.svc.cluster.local:3306
认证:  root / password123
Auth插件: mysql_native_password（ConfigMap: mysql-config 强制设置，避免 asyncmy 1045）
PVC:   mysql-data（持久化）
宿主机访问: kubectl port-forward svc/mysql -n mdp 3307:3306  → 127.0.0.1:3307
⚠️  重要: OrbStack 会缓存旧握手包，必须用 port-forward 3307，不能直接用 3306
```

### 实时分析 — Apache Doris（OLAP）

```
用途:  实时分析引擎 + 按需检索时的物化视图查询
Pod:   doris-fe-0（Frontend）、doris-be-0（Backend）
FE地址: doris-fe.mdp.svc.cluster.local:8030（HTTP）/ :9030（thrift）
BE地址: doris-be.mdp.svc.cluster.local:9060
使用场景:
  - Object 实例数据的聚合分析查询
  - 物化视图加速"按需检索"（on-demand retrieval）
  - 替代 Spark 做低延迟批量计算
```

### 消息总线 — Apache Kafka

```
用途:  系统内所有异步事件传递（Control → Data Plane 的解耦核心）
Pod:   kafka-0（StatefulSet）
Broker: kafka.mdp.svc.cluster.local:9092（集群内）
         localhost:9094（NodePort 外部访问）
Schema Registry: schema-registry（当前状态：CrashLoopBackOff，Avro 功能受影响）
Kafka Connect: kafka-connect（1 副本，Running）
关键 Topics（规约）:
  - mdp.ontology.events     对象类型定义变更事件
  - mdp.object.data.raw     原始对象数据摄入
  - mdp.index.requests      索引任务请求
```

### 全文索引 — Elasticsearch

```
用途:  全文检索加速（对象属性、文档内容、元数据搜索）
Pod:   elasticsearch-0（StatefulSet）
地址:  elasticsearch.mdp.svc.cluster.local:9200
当前状态: Running（正常）
使用场景:
  - 对象实例的全文属性检索
  - DataCatalog 的表/列名搜索
  - 非结构化内容的关键词召回
```

### 向量存储 — Milvus

```
用途:  向量数据存储（嵌入式检索、语义相似度搜索）
Pod:   milvus-*（当前状态：Error，15次重启）
依赖:  milvus-etcd-*（Running）、milvus-minio-*（Running）
当前状态: ⚠️  Milvus 主 Pod 异常，向量检索功能降级
使用场景（计划）:
  - ObjectProperty VECTOR 类型的存储和检索
  - 语义相似度的对象关联发现
备注: 向量功能由 ENABLE_VECTOR_SEARCH Feature Flag 控制，降级时自动跳过
```

### 对象存储 — MinIO（原始数据存储）

```
用途:  原始数据文件持久化（DataCatalog 的原始数据层）
Pod:   minio-*（Running）
地址:  minio.mdp.svc.cluster.local:9000
Bucket 命名惯例: mdp-raw-{connector_id}
独立MinIO（非Milvus）: minio-6cf89dbc66（Running）
Milvus专用MinIO: milvus-minio-*（Running）
使用场景:
  - Ingestion Worker 的原始文件落地（S3 兼容接口）
  - Iceberg 表的底层存储层（Lakehouse）
```

### 缓存 — Redis

```
用途:  系统缓存（元数据读缓存、Leader Election、Scenario 沙箱数据）
Pod:   redis-*（Running）
地址:  redis.mdp.svc.cluster.local:6379
使用场景:
  - @cached 装饰器：元数据 API 读缓存（TTL 300s）
  - RedisLeaderElection：调度器主节点选举（SET NX EX）
  - Scenario Overlay：What-If 场景的 ObjectEdit 快照缓存
```

## WHEN（触发条件 / 危险信号）

加载此记忆的场景：
- 🔴 任何涉及 DB 连接、Kafka 发送、ES 索引的 EP
- 🔴 部署相关任务：镜像构建、滚动更新、port-forward
- 🟡 调试连接问题（1045 / 连接拒绝 / Timeout）
- 🟡 设计新数据流（哪条链路用 Kafka / ES / Doris / Milvus）

危险信号：
- ❌ 直连 3306 端口（应用 3307 port-forward）
- ❌ 假设 Schema Registry 正常（当前 CrashLoopBackOff）
- ❌ 依赖 Milvus（当前 Error 状态，需检查 Feature Flag）
- ❌ 在 Service 层直接 import pymilvus（必须走 infrastructure 适配器）
