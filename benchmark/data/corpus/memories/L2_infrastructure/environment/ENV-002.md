# ENV-002 · 宿主机测试数据源配置（MySQL + PostgreSQL）

## WHERE（适用场景）

- 开发 / 测试数据连接器（Connector）功能时
- 写集成测试需要外部数据源时
- 配置 SyncJob 指向宿主机数据库时
- 排查 Connector "连接失败" 问题时

## HOW（核心配置）

### 宿主机 MySQL（测试数据源）

```
用途:  DataLink 连接器集成测试数据源（非元数据库）
Host:  host.docker.internal（Docker Compose 内访问宿主机）
       宿主机可达 IP（K8s Pod 访问宿主机）
Port:  3306
账号:  宿主机 MySQL 实际账号（不能假设 root/password123）
⚠️  与 K8s 元数据库隔离：宿主机 3306 ≠ K8s mysql:3306
用途区分:
  - K8s mysql:3306  → MDP 元数据（meta_* / sys_*），不作为 Connector 数据源
  - 宿主机 mysql:3306 → 外部业务数据源，用于 Connector + SyncJob 测试
```

### 宿主机 PostgreSQL（测试数据源）

```
用途:  DataLink 连接器集成测试数据源（非元数据库）
Host:  host.docker.internal（Docker Compose 内）
       宿主机可达 IP（K8s Pod 内）
Port:  5432（默认）
账号:  postgres / <宿主机实际密码>
数据库: mdp_source_pg（建议用于测试）
```

### Connector 配置规范

当在 MDP 中配置 Connector 指向宿主机数据源时：

```python
# ✅ 正确：使用 host.docker.internal 或实际 IP
connector_config = {
    "host": "host.docker.internal",  # Docker Compose 场景
    "port": 3306,
    "database": "mdp_source_mysql",
    "username": "...",
}

# ❌ 错误：127.0.0.1/localhost 在 K8s Pod 内会命中 MDP_DATA_SOURCE_HOST_ALIAS
# 被替换为 mysql.mdp.svc.cluster.local，连接的是元数据库！
connector_config = {
    "host": "127.0.0.1",  # 危险：K8s 内会被替换为元数据库地址
}
```

### 后端别名注意事项

```
环境变量: MDP_DATA_SOURCE_HOST_ALIAS=mysql.mdp.svc.cluster.local
行为: 当 Connector.host 为 127.0.0.1/localhost 时，后端自动替换为该地址
影响: 若想连宿主机数据库，必须使用 host.docker.internal 或真实宿主机 IP
```

## WHEN（触发条件）

- 🔴 开发 Connector 功能、SyncJob 配置时必读
- 🟡 调试 "1045 Access denied" 或 "Connection refused"：先区分是元库还是外部数据源
- 🟡 写 DataLink 集成测试的 fixture 时

地址速查：

| 场景 | 元数据库（K8s） | 外部数据源（宿主机） |
|:---|:---|:---|
| K8s Pod 内部 | `mysql:3306` | `host.docker.internal:3306` |
| 宿主机 port-forward | `127.0.0.1:3307` | `127.0.0.1:3306`（直连） |
