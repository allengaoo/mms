# ADR-001: Hexagonal Architecture (Ports & Adapters)

**ID**: AD-001
**Type**: Architecture Decision
**Tags**: [architecture, hexagonal, ports-adapters, dependency-inversion]
**Status**: accepted

## Decision

Business logic must NOT import infrastructure libraries directly.
Infrastructure access must go through `app/infrastructure/*` adapters.

## Rationale

- Enables unit testing without spinning up real databases/queues
- Infrastructure can be swapped without touching business logic
- Prevents accidental coupling between layers

## Rules

- ✅ Services import from `app/infrastructure/db`, `app/infrastructure/cache`
- ❌ Services import `pymilvus`, `aiokafka`, `elasticsearch` directly
- ❌ Services import `sqlalchemy` for raw queries (use repository pattern)

## Exception

ORM model definitions (`sqlmodel.Field`, `Column`) are allowed in the domain layer since they describe the schema, not infrastructure behavior.
