# AD-002 · Row-Level Security 基于 tenant_id（架构基线）

## 决策
多租户隔离通过应用层 RLS 实现（所有查询加 `WHERE tenant_id = ctx.tenant_id`），不使用数据库级 RLS。

## 背景
MySQL 8.0 没有原生行级安全，实现复杂；应用层 RLS 更灵活，可在审计日志中记录 tenant 上下文。

## 影响
- ❌ 遗漏 tenant_id 过滤构成 IDOR（越权访问）安全漏洞，这是红线
- ✅ 每个 Repository 方法必须包含 `.where(Model.tenant_id == ctx.tenant_id)`
- ✅ `global-constraints.mdc` 第 2 条红线强制执行
