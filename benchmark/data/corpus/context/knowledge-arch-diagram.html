// 优化逻辑 · 为什么这样设计
P0 · 最高优先级
移除 3 个错位的 alwaysApply
post-task.mdc 是「EP 末尾才用」的工具，却用 alwaysApply 在每条消息消耗 2,700t。 这相当于每次让模型读一本手册，然后问「今天天气怎样」。 context_opt 的 Scope 表被 8 个 Manifest 完整替代。 doc_maintainer 的触发表已存在于 constraints #15-16。
效果：-4,000t/次，立即生效
P1 · 高优先级
Model 路由 + Compact 触发
小模型（Kimi2.5 等）在 32k 窗口内如果基础消耗就占 4,489t， 实际可用只剩 27k，且会被无关审计规则「污染」注意力。 优化后基础消耗降到 500t，配合任务复杂度→模型路由， 简单 bug 修复用 fast model 可节省 10x 成本。
效果：fast model 可处理 ~70% EP 任务
P2 · 中优先级（可选）
LESSONS_LEARNED Instinct 格式
当前 LESSONS_LEARNED 每条约 150 token（叙述段落格式）。 Instinct 格式（条件 + 动作 + 禁止，每条 ~40t） 在同等信息密度下节省 60% 解析开销， 且更易被小模型准确执行（指令格式 vs 叙述格式）。
效果：加载记忆层时节省 60%
